Present a complete implementation plan for explicit user approval. This
is the only approval mechanism in plan mode — never ask "should I
proceed?" in text or through ask_user.

A good plan contains the recommended approach only: the files to change,
the existing code to reuse (with paths), the order of work, and how the
result will be verified. Skimmable but executable — no filler steps and
no survey of rejected alternatives. Do not use this for research or
question-answering tasks that end in an answer rather than
implementation.
