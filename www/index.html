▚ 01 DIAGNOSTICS

The build loop is the agent loop

Compiler errors and warnings stream in over Dune RPC as they happen. The agent sees the fallout of its own edit before the next step — a clean build, not "the edit applied", is the baseline for done.

▚ 02 TOOLING

Merlin, not grep

Where a generic agent greps and guesses, Spice asks Merlin where a value is defined, asks Dune what the project looks like, and reads API surfaces instead of whole files.

▚ 03 SAFETY

Safe by default

Durable permission rules, sandboxed execution that fails closed, workspace trust, and per-turn session diff and session revert.

▚ REVIEW

Code review lives in the source

Spice speaks the CR review-comment convention. Drop a comment in the code —

let normalize path =
(* CR spice: this validation belongs in Spice_path, not here *)
if String.contains path ';' then invalid_arg "path";

— and the workspace watcher delivers it to the agent live, mid-session: feedback anchored to the exact code it is about. Addressed comments resolve as XCR; CR-soon defers work without losing it.

▚ TOOLS

Tools that understand OCaml

Alongside the usual file, search, and shell tools, the model gets OCaml-native tools.

ocaml_dune_describe

The project as Dune sees it: libraries, executables, dependencies, tests.

ocaml_docs

API surfaces by name, across workspace libraries and locked dependencies.

ocaml_find_definitions

Identity-based navigation through Merlin, not textual grep. Paired with ocaml_find_references.

ocaml_ast_edit

Edits addressed by compiler AST location; fragments that don't parse are rejected.

ocaml_eval

Toplevel phrases evaluated in the project's own context.

ocaml_dune_diagnostics

The current compiler and Dune error set, on demand.

▚ WHY OCAML

A language built for machine-checkable correctness

Specialization is only half of the story. OCaml is the strongest target for building the safest, most productive coding agent there is.

01 · feedback

A sound type system and explicit module interfaces rule out whole categories of bugs before the agent reaches the test suite.

02 · ecosystem

Typed build rules, property-based testing, documentation-as-contract: rich verification loops the agent works inside.

03 · verification

Rocq's deep OCaml integration points beyond "compiles and passes tests", toward formally verified code where it matters.
