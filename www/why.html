▚ THE ARGUMENT

Why Spice?

The coding agent era demands a new foundation for software development.

01 · THE PROBLEM

AI agents are heading toward full autonomy — writing, testing, and shipping software without supervision. For that to work, we need to trust the code they produce.

Today, more AI-written code means more bugs and more technical debt. Review and testing become the bottleneck: human developers fixing agent output instead of building. Linear answers to an exponential problem.

02 · WHAT YOU GET TODAY

If you write OCaml, Spice is the best agent you can point at your codebase today — we use it on Spice itself, daily.

General-purpose agents waste cycles on language quirks and tooling confusion. Spice understands the language and its ecosystem: where a generic agent greps and guesses, it asks Merlin where a value is defined, asks Dune what the project looks like, fetches documentation on demand, and gets real-time diagnostics as it generates code. In our use, it converges faster and produces higher-quality work.

The agent is only as good as the feedback loops it can rely on. So we're also building them: windtrap for testing, thumper for benchmarking, a linter, and eventually formal verification tooling — all designed for AI-first usage. Beyond tooling: OCaml-trained models, AST-based code generation, type-guided synthesis.

03 · WHY OCAML

Why build here, when most of the world's code isn't OCaml? Because correctness has to be machine-checkable before it can be automated — and OCaml is the strongest available combination of guarantees.

Its type system eliminates entire categories of bugs: no type errors, no memory corruption, no segfaults. With OxCaml's data-race-free parallelism, race conditions go too. QCheck property-based testing and Rocq give a credible path toward provably correct code.

We make AI-built software correct so it can become autonomous — agents collaborating on large codebases while you focus on what to build, not how to fix it.

04 · FREE TOOLS

Compilers, editors, linters — the best developer tools have always been free. But as coding agents widen the productivity gap, we're heading toward a future where you need a $200/month subscription to stay competitive. That's not a future we want to live in.

Spice is free and always will be. Use any LLM provider — Claude, GPT, Gemini, or local models. No vendor lock-in, no monthly fees. Just like your compiler, Spice belongs in every developer's toolkit, accessible to all.

05 · THE STACK

Spice is the agent layer of Invariant's stack — alongside models fine-tuned for OCaml and correctness-sensitive development, and a runtime that deploys models as self-contained binaries, close to the hardware.

The endgame: you describe what to build, and you can trust what comes back — even as the codebase grows for years under agents. That takes a foundation that prevents bugs by design. It starts with OCaml, and with Spice.

status: experimental — the core loop works, but interfaces and formats will change without notice. Expect rough edges; the foundation is the bet.
